IBITIcoin — Whitepaper
A top-standard structure (Abstract → Definitions → Architecture → Security/Risks → Roadmap). Plain English, no hype.
Network: BNB Smart Chain (chainId 56) • Token: IBITI (8 decimals)

Abstract

IBITIcoin is a modular on-chain ecosystem on BNB Smart Chain: an IBITI token, staking with time-based rewards, NFT utility/discount layers, a DAO module, a buyback manager, and a website purchase flow. The website purchase flow uses a verified UUPS Proxy router to forward USDT to a reserve wallet and distribute IBITI rewards from a funded inventory.

This document is written for normal users and reviewers: it explains the “why” in plain language, and the “how” with concrete on-chain mechanics.

1) Problem & Goals

Problem

Many tokens have unclear funds flow, vague utilities, and “trust me” mechanics. Reviewers (and users) want: clear roles, clear contracts, clear risks, and clear on-chain proof.

Goals
  • Transparent on-chain modules with explicit ownership & pausing.
  • Website buy flow where USDT is forwarded immediately (router holds ~0 USDT).
  • Staking that is simple: choose period → earn reward (and sometimes NFT rewards).
  • NFT utilities that are concrete (discount NFTs with expiry + special Pandora behavior).

2) Definitions (quick)

  • Reserve Wallet: receives USDT from website purchases; funds operations/liquidity.
  • Promo Router (UUPS): website purchase contract behind a proxy; can be upgraded by owner for critical fixes.
  • Staking duration: 1–12 months (1 month = 30 days in contract).
  • Grace period: extra time after unlock to claim (defined in staking).
  • Discount NFT: ERC-721 “DiscountNFT” that can be used (burned) for a discount; expires by level if unused.

4) Website purchase flow (Cash Promo + Referral, UUPS)

This is the “clean” part: user pays USDT, USDT is forwarded to reserve wallet in the same transaction, IBITI rewards are paid from router inventory.

Money flow (one transaction)

User USDT → Router (transferFrom) → Reserve Wallet (transfer)
Router → Buyer: IBITI base + promo bonus (bonusBps)
Router → Referrer: fixed IBITI (refReward) if eligible

Price quoting

Router reads reserves from the IBITI/USDT Pancake pair and calculates output. Two quote modes exist:

Quote mode 0 — AMM-style (recommended)

Uses reserve math similar to Pancake/Uniswap V2 output, including feeBps (e.g. 25 bps). Larger buys naturally produce less favorable price (slippage), which is safer.

Quote mode 1 — spot reserve ratio (not recommended)

Uses spot reserve ratio. It ignores slippage and is more sensitive to low-liquidity conditions.

Protection & limits

  • Buy limits: minPaymentAmount / maxPaymentAmount.
  • Slippage guard: buyer supplies minIbitiOut, tx reverts if quote is smaller.
  • Spot bounds (optional): minSpotPrice / maxSpotPrice.
  • Pause & critical updates: requirePauseForCritical can require pause before critical config changes.
Promo math (simple)

bought = quote(paymentUSDT)
bonus = bought × bonusBps / 10000
toBuyer = bought + bonus
toReferrer = refReward (if eligible)

5) Staking — periods, rewards, and what happens if you ignore your unlock

Staking is time-based: you choose a duration (1–12 months). Rewards are percentage-based and known in advance. Rewards can include bonus NFTs (configured for specific durations).

How duration works

  • Duration is 1 to 12 months (on-chain rule).
  • 1 month = 30 days (contract constant).
  • After maturity, there is a grace period of 180 days to claim normally.
  • If you miss maturity + 180 days, stake is treated as expired: principal is sent to the treasury (forfeited).

Funding & payout mechanics (for reviewers)
  • Staking contract holds staked IBITI.
  • On unstake, payout = principal + reward − penalty (depending on timing).
  • If staking contract balance is not enough, it can pull IBITI from treasury via transferFrom (requires treasury allowance).
  • NFT rewards are minted via NFTDiscount.mintJackpot() (and failures do not revert unstake; they’re caught).

6) NFTs — two layers (IBITINFT + DiscountNFT)

A) IBITINFT (collectible/utility NFT)

  • Minted via purchase functions: purchaseNFT(tokenURI) (paid in IBITI) and purchaseNFTWithUSDT(tokenURI) (paid in USDT).
  • Payment is transferred directly to contract owner.
  • Unique URI enforced: the same tokenURI cannot be reused.
  • Monthly price update: if monthly sales ≥ salesThreshold, IBITI price grows by priceGrowthRate (basis points).

B) DiscountNFT (NFTDiscount contract)

This is a dedicated ERC-721 for discounts. Most discount NFTs are single-use and burn on use. Many also expire if unused (depends on level).

  • Pandora minting is restricted and not issued via jackpot mint.

7) Security & Trust Model (straight talk)

  • UUPS promo router is upgradeable — this is a conscious tradeoff for patching critical bugs fast. Upgrades are on-chain and visible.
  • Most other modules are deployed as standard contracts (non-proxy) unless explicitly proxied.
  • Pausable patterns exist to stop the system during emergencies.
  • Price reference risk: website quote uses Pancake pair reserves → can be manipulated in low liquidity. Mitigations exist (limits, minOut, optional spot bounds).

8) Risks & limitations

This section exists so reviewers don’t have to guess. And so users don’t get wrecked by surprise rules.

  • Market risk: volatility and liquidity depth can change fast.
  • Reserve manipulation: pair-based quotes can be attacked; use minIbitiOut and keep buy limits.
  • Staking expiry rule (important): if you ignore unlock + grace (180 days), principal is forfeited to treasury.
  • Contract risk: bugs or integration mistakes can still happen (DeFi reality).
  • Admin risk: owner can change parameters where contracts allow it; actions are on-chain.

9) Roadmap (high level)

  • Now: liquidity growth, stable website buy flow, transparent dashboards.
  • Next: expand NFT utilities + staking UI, community voting experiments.
  • Later: governance hardening (multisig), broader integrations, additional dApps.

10) Disclaimer

  • IBITI is a utility/governance token for an experimental ecosystem. Not equity, not debt.
  • No profit, dividends, or guaranteed buybacks are promised.
  • This is not financial advice. Users are responsible for compliance and taxes.
