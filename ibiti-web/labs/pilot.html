Safe Execution for AI Agents

IBITI EPK is an on-chain execution kernel that enforces hard permissions for autonomous agents: spend limits, allowlists/deny lists, typed intents (EIP-712), nonces/deadlines, and instant revoke. No “trust the bot”. Only deterministic constraints.

EIP-712 intents Modular validators Instant revoke (panic)
The problem

AI is probabilistic. Money is not.

Agents hallucinate, get prompt-injected, or mis-execute trades. If an agent holds a key or unlimited approvals, a single mistake becomes irreversible loss. Protocols and platforms cannot put user funds behind “best effort”.

  • • Key compromise becomes total loss.
  • • “Unlimited approve” turns into unlimited damage.
  • • Off-chain guardrails fail when the on-chain call is allowed.
The fix

Deterministic constraints on-chain.

EPK enforces policy in execution: spend caps per tx and rolling windows, allow/deny targets and call keys, EIP-712 signatures (human-auditable intent), plus nonce/deadline replay protection. If policy fails — the transaction reverts. No UI promises. No “trust me”.

  • • Spend limits enforced in the kernel, not in UI.
  • • Whitelist / denylist for targets and call signatures.
  • • Instant revoke (panic) to halt agent execution.
