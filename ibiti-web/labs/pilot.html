WHY NOW

Agents are here. Trust is missing.

On-chain AI agents can execute fast — but wallets still rely on signatures humans can’t audit. That gap becomes the #1 risk surface: unlimited approvals, replay, stealth calldata, and irreversible loss.

THE PROBLEM
  • • Users sign “something”, not intent. Most approvals are blind.
  • • Revocation is slow and fragmented (multiple approvals, multiple dapps).
  • • Risk controls are app-specific — not enforced at the execution layer.
  • • Typical “security” = marketing. Real enforcement is missing.
THE SOLUTION

Eternal Permission Kernel (EPK)

An immutable execution kernel that gates every action through auditable intent + optional validator modules. If rules fail — execution reverts atomically. If emergency — revoke instantly.

immutable EIP-712 revocable
IMMUTABLE CORE
No proxy admin. No upgrades. No “oops we changed security later”. The kernel is frozen.
MODULAR VALIDATORS
Spend limits, allowlists, time windows, rolling windows, multi-factor, session policies — as modules.
INSTANT REVOCATION
One action disables agent execution. Emergency switch is a feature, not a hope.
