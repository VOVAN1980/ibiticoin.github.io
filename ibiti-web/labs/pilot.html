The problem

AI is probabilistic. Money is not.

LLMs hallucinate, agents get prompt-injected, bots mis-execute trades. If an agent holds keys or unlimited approvals, one mistake becomes irreversible loss.

The missing piece is deterministic execution constraints — enforced on-chain.
The fix

Policy-first execution on-chain.

EPK verifies intent (EIP-712), checks nonce/deadline, applies policy limits and optional validator rules, then executes atomically. If anything violates the policy — it reverts. No “trust me bro”.

  • • Spend limits (per-tx / rolling window)
  • • Allowlist / denylist targets + calldata constraints
  • • Instant revoke (panic) + TTL / expiries
